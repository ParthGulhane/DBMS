CREATE TABLE Department (
    dept_id INT PRIMARY KEY,

    emp_id INT PRIMARY KEY,
    emp_name VARCHAR(50),
    salary INT,
    dept_id INT,
    FOREIGN KEY (dept_id) REFERENCES Department(dept_id)
);

✅ 3. Subqueries (Main Part)
🔹 1. Employees with salary above average
SQL
SELECT emp_name, salary
FROM Employee
WHERE salary > (
    SELECT AVG(salary) FROM Employee
);
🔹 2. Employees working in IT department
SQL
SELECT emp_name
FROM Employee
WHERE dept_id = (
    SELECT dept_id FROM Department WHERE dept_name = 'IT'
);
🔹 3. Employees with highest salary
SQL
SELECT emp_name, salary
FROM Employee
WHERE salary = (
    SELECT MAX(salary) FROM Employee
);
🔹 4. Employees in departments with more than 1 employee
SQL
SELECT emp_name
FROM Employee
WHERE dept_id IN (
    SELECT dept_id
    FROM Employee
    GROUP BY dept_id
    HAVING COUNT(*) > 1
);
🔹 5. Departments having employees with salary > 6000
SQL
SELECT dept_name
FROM Department
WHERE dept_id IN (
    SELECT dept_id
    FROM Employee
    WHERE salary > 6000
);

THEORY

✅ Aim
To write and execute subqueries in SQL to retrieve data from one table based on the results of another table.
✅ Theory
A Subquery is a query written inside another SQL query. It is used to perform operations that depend on the result of another query.
Subqueries are enclosed in parentheses () and are commonly used in clauses like:
WHERE
FROM
SELECT
🔹 Types of Subqueries
Single-row subquery → returns one value
Multi-row subquery → returns multiple values
Correlated subquery → depends on outer query
🔹 Uses of Subqueries
Compare values (>, <, =)
Find aggregate values (AVG, MAX, MIN)
Filter data based on another table
Work with IN, EXISTS, ANY, ALL
✅ Syntax
🔹 Basic Subquery
SQL
SELECT column_name
FROM table_name
WHERE column_name operator (
    SELECT column_name
    FROM table_name
    WHERE condition
);
🔹 Subquery using IN
SQL
SELECT column_name
FROM table_name
WHERE column_name IN (
    SELECT column_name
    FROM another_table
);
🔹 Subquery using EXISTS
SQL
SELECT column_name
FROM table_name
WHERE EXISTS (
    SELECT column_name
    FROM another_table
    WHERE condition
);
🔹 Subquery with Aggregate Function
SQL
SELECT column_name
FROM table_name
WHERE column_name > (
    SELECT AVG(column_name)
    FROM table_name
);
✅ Conclusion
Thus, subqueries were successfully implemented to retrieve data based on results from another query. Subqueries help in performing complex queries efficiently and improve the flexibility of SQL operations.


VIVA

✅ Subqueries Viva Answers
🔹 What is a Subquery?
A subquery is a query written inside another SQL query, used to retrieve data based on the result of another query.
🔹 What is a Correlated Subquery?
A correlated subquery depends on the outer query and is executed once for each row of the outer query.
👉 Example idea: “For each employee, compare with their department data”
🔹 What is a Derived Table?
A derived table is a subquery written in the FROM clause and treated as a temporary table.
