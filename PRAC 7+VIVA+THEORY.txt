CREATE PROCEDURE getAllStudents()

    DECLARE s_name VARCHAR(50);

        SELECT name FROM Student;

    );

    OPEN cur;

        FETCH cur INTO s_name;

        IF done THEN
            LEAVE read_loop;
        END IF;

    END LOOP;

    -- Close cursor
    CLOSE cur;

    -- Show result
    SELECT * FROM temp_students;
END //

Q: What is a stored procedure?

cal - AISSMS College of Engineering, Pune

→ A named, precompiled block of SQL stored in the database. Executed using CALL. Accepts IN, OUT, INOUT parameters. Runs on the server - reduces network traffic.

Q: Difference: procedure vs function

Function MUST return a value and can be used in SELECT. Procedure returns nothing directly (uses OUT params) and is called with CALL. Procedures can have transactions; functions cannot.

Q: What is DELIMITER and why do we use it?

→ MySQL uses; to end statements. A procedure body contains multiple; lines. We change the delimiter to // so MySQL treats the entire procedure as one unit until //, then restore to;.

Q: What is a cursor?

→ A pointer to a row in a query result set. Allows row-by-row processing inside a procedure. Steps: DECLARE → OPEN FETCH in LOOP CLOSE.

Q: What is NOT FOUND handler?

DECLARE CONTINUE HANDLER FOR NOT FOUND fires when FETCH finds no more rows. We set a done flag = 1 and LEAVE the loop. Without this the procedure would error at the last FETCH.

Q: What does DETERMINISTIC mean in a function?

→ Tells MySQL the function always returns the same result for the same input. Required for some replication setups. Opposite is NOT DETERMINISTIC (result can vary - e.g. uses NOW() or RAND ()).

Q: IN vs OUT vs INOUT?

IN: value passed in, cannot be modified. OUT: variable receives value from procedure. INOUT: passes a value in, procedure modifies and returns it.


*Always restore DELIMITER to; after creating the procedure.

* DECLARE statements must appear BEFORE any executable code inside BEGIN...END. 
*Cursor result set is static - fetches the rows as they were when OPEN was called. 
*Cursors are forward-only - you can only FETCH the next row, never go back.
* For OUT params: CALL proc(arg, @var); then SELECT @var; to see the result. 
*DROP PROCEDURE IF EXISTS avoids errors if the procedure doesn't exist yet.
